# Notes

Personal learning log + project memory for `eudora-pantry-site`.

Each entry is a discrete, retrievable chunk — RAG-style. The goal is that any single file makes sense on its own without reading the rest of the folder, so an LLM (or future-you) can grab one note and have full context.

## Conventions

- **Filename:** `YYYY-MM-DD-short-topic.md`
- **Frontmatter:** every entry starts with YAML — `date`, `tags`, `status` (shipped / in-progress / abandoned), `topic`
- **Self-contained:** include the *why* and the *what next*, not just the *what*. A future search hit on the title should give the reader the full picture.
- **Tag freely:** `design`, `accessibility`, `mvp`, `sanity`, `payments`, `deploy`, `seo`, `decision`, `learning`
